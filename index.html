REQ-001 오류상황판 행 선택 오류상황판 각 행의 첫 번째 컬럼에는 이벤트 조치 대상을 선택할 수 있는 체크박스를 제공한다. 적용
REQ-002 오류상황판 선택 제한 오류상황판에서는 같은 거래코드와 에러코드를 가진 행만 동시에 여러 건 선택할 수 있으며, 조건이 다른 행은 선택할 수 없도록 비활성화한다. 적용
REQ-003 오류상황판 이벤트 조치 선택된 오류상황판 행은 추후 이벤트 조치 대상이 된다. 대기
REQ-004 오류상황판 이벤트 조치 이벤트 조치하기 실행 시 선택된 오류상황판 행이 없으면 조치 대상 선택 안내 팝업을 표시한다. 적용
REQ-005 오류상황판 오류 상세 오류 상세 페이지는 상세내용과 이벤트 조치접수 탭으로 구분한다. 적용
REQ-006 오류상황판 오류 상세 오류상황판 행을 클릭하면 상세 페이지를 열고 상세내용 탭을 기본 표시한다. 적용
REQ-007 오류상황판 이벤트 조치 이벤트 조치하기 실행 시 선택된 오류상황판 행의 상세 페이지를 열고 이벤트 조치접수 탭으로 바로 이동한다. 적용
REQ-008 오류상황판 이벤트 조치 이벤트 조치접수의 조치담당자는 직접 입력할 수 없으며 돋보기 검색을 통해서만 선택한다. 적용
REQ-009 오류상황판 이벤트 조치 조치담당자 검색 결과는 SWING, ITBPI 등 인사정보 데이터에서 가져온 정보만 사용한다. 대기
REQ-010 오류상황판 이벤트 조치 조치 예정일과 조치완료일은 직접 입력하지 않고 달력에서 유효한 날짜를 선택한다. 적용
REQ-011 오류상황판 이벤트 조치 여러 오류상황판 행을 선택해 이벤트 조치하기를 실행한 경우 상세내용 탭은 표시하지 않는다. 적용
REQ-012 오류상황판 이벤트 조치 이벤트 조치접수는 조치담당자, 조치 예정일, 조치구분, 조치내용을 입력해 저장한 뒤 조치완료일을 입력할 수 있으며, 저장 이후 조치 예정일은 수정할 수 없고 조치담당자, 조치구분, 조치내용은 계속 수정할 수 있다. 적용
REQ-013 오류상황판 이벤트 조치 이벤트 조치접수에서는 조치 예정일 입력 단계에 저장 버튼을 표시하고, 조치완료일 입력 단계에 결재요청 버튼을 표시한다. 적용
REQ-014 오류상황판 이벤트 조치 결재요청 버튼 실행 시 담당 셀장 결재를 위해 SWING, ITBPI 등 인사정보 데이터에서 담당 셀장 정보를 조회한다. 대기
REQ-015 오류상황판 이벤트 조치 결재요청 버튼 실행 시 결재자에게 SWING 메일을 전송하며, 메일 내용은 추후 협의 예정이다. 대기
REQ-016 오류상황판 이벤트 조치 이벤트 조치접수 저장 시 조치담당자, 조치구분, 조치 예정일, 조치내용 중 하나라도 입력되지 않으면 저장하지 않고 입력 안내를 표시한다. 적용
REQ-017 오류상황판 이벤트 조치 조치 예정일은 과거 날짜를 선택할 수 없으며 오늘 또는 이후 날짜만 선택할 수 있다. 적용
REQ-018 오류상황판 이벤트 조치 조치완료일은 과거 날짜를 선택할 수 없으며 오늘 또는 이후 날짜만 선택할 수 있다. 적용
REQ-019 오류상황판 이벤트 조치 결재요청 실행 시 조치완료일, 조치담당자, 조치구분, 조치내용 중 하나라도 입력되지 않으면 결재요청하지 않고 입력 안내를 표시한다. 적용
REQ-020 이벤트관리 메뉴 이벤트 관리 탭을 클릭하면 하위 메뉴로 이벤트 조회와 이벤트 룰 관리를 표시한다. 적용
REQ-021 이벤트관리 이벤트 조회 이벤트 조회 페이지 상단 검색 영역은 오류구분, 조치구분, 발생일시 시작일자와 종료일자, 담당부서, 담당자, 서비스명, 진행상태 조건을 제공하며, 담당부서와 담당자는 돋보기 검색으로만 선택하고 SWING 또는 ITBPI 정보를 사용한다. 서비스명은 거래코드에서 뒤 2자리를 제외한 값으로 조회하며, 진행상태는 미조치를 기본값으로 한다. 적용
REQ-022 이벤트관리 이벤트 조회 이벤트 조회 검색 조건은 필수 입력 항목 없이 선택적으로 입력하며, 조건을 입력하지 않아도 조회할 수 있다. 적용
