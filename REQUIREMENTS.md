# ITSM MVP 개발요건

## 관리 원칙

- 사용자가 기능적으로 설명한 요건만 기록한다.
- 샘플 데이터 형식, 구현 방식, 배포 방식 같은 기술 세부사항은 개발요건 표에 넣지 않는다.
- 사용자가 명시적으로 "개발요건에 추가해줘"라고 요청한 항목만 화면의 `개발요건` 탭과 이 `REQUIREMENTS.md` 파일에 함께 누적한다.

## 요구사항 목록

| ID | 구분 | 요건 | 상태 |
| --- | --- | --- | --- |
| REQ-001 | 행 선택 | 오류상황판 각 행의 첫 번째 컬럼에는 이벤트 조치 대상을 선택할 수 있는 체크박스를 제공한다. | 적용 |
| REQ-002 | 선택 제한 | 오류상황판에서는 같은 거래코드와 에러코드를 가진 행만 동시에 여러 건 선택할 수 있다. | 적용 |
| REQ-003 | 이벤트 조치 | 선택된 오류상황판 행은 추후 이벤트 조치 대상이 된다. | 대기 |
| REQ-004 | 이벤트 조치 | 이벤트 조치하기 실행 시 선택된 오류상황판 행이 없으면 조치 대상 선택 안내 팝업을 표시한다. | 적용 |
