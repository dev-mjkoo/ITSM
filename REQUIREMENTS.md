# ITSM MVP 개발요건

## 관리 원칙

- REQ-000: 신규 개발요건은 화면의 `개발요건` 탭과 이 `REQUIREMENTS.md` 파일에 함께 누적한다.

## 요구사항 목록

| ID | 구분 | 요건 | 상태 |
| --- | --- | --- | --- |
| REQ-001 | 배포 | HTML, CSS, JavaScript 파일만으로 로컬에서 실행 가능한 MVP 시연본으로 구성한다. | 적용 |
| REQ-002 | 화면 | 좌측 메뉴에는 오류상황판, 이벤트 관리, 개발요건 탭을 제공한다. | 적용 |
| REQ-003 | 오류상황판 | 오류상황판은 선택, NO, 등록시간, 채널유형, 파트, 최종이행자, 담당자, 거래코드, 거래명, 오류구분, 에러코드, 에러메시지, 거래점, 호스트명, PID 컬럼을 제공한다. | 적용 |
| REQ-004 | 행 선택 | 오류상황판 첫 컬럼에는 체크박스를 제공하고, 같은 거래코드와 에러코드를 가진 행만 동시에 여러 건 선택할 수 있다. | 적용 |
| REQ-005 | 이벤트 조치 | 선택된 오류상황판 행은 추후 이벤트 조치 대상이 된다. | 대기 |
| REQ-006 | 샘플 데이터 | 샘플 데이터는 `sample-data.js`로 분리하고, 로컬 파일 실행에서도 읽히도록 `script` 방식으로 연결한다. | 적용 |
| REQ-007 | 데이터 규칙 | 채널유형은 `A1`, `A0`, `DT`, `D0`, `DX`를 사용하며 D 계열은 `BXM` 오류, A 계열은 `PFM` 오류로 구분한다. | 적용 |
| REQ-008 | 데이터 규칙 | D 계열 거래코드는 `RSMDE0000A01` 형식, A 계열 거래코드는 `GSSDD0000A01` 또는 `GSTYJ0000A01` 형식을 사용한다. | 적용 |
| REQ-009 | 데이터 규칙 | D 계열 파트는 `디지털수신`, `모바일라이프`, `마이데이터`, `디지털여신` 값을 섞어서 사용한다. | 적용 |
| REQ-010 | 데이터 규칙 | A 계열 호스트명은 `coreap1p`부터 `coreap4p`, D 계열 호스트명은 `solwa01p`부터 `solwa05p` 또는 `dtnwa21p`부터 `dtnwa25p`를 사용한다. | 적용 |
| REQ-011 | 데이터 규칙 | 거래점은 한글 지점명이 아니라 1000부터 7999 사이의 4자리 숫자 값을 사용한다. | 적용 |
